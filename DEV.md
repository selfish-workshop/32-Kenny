# InterLink — 개발 개요

## 서비스 아키텍처

```
[사용자 브라우저]
      ↓
[Frontend: React]
      ↓
[Backend API: Node.js / Express]
      ↓
[Database: PostgreSQL]
      ↓
[외부 연동: 결제 PG, 이메일/알림]
```

## 기술 스택

| 영역 | 기술 |
|------|------|
| Frontend | React, Tailwind CSS |
| Backend | Node.js, Express |
| Database | PostgreSQL |
| 인증 | JWT (기업 / 통역사 구분) |
| 결제 | Stripe 또는 토스페이먼츠 |
| 배포 | Vercel (FE) + AWS EC2 (BE) |

## 주요 모듈

### 1. 사용자 인증
- 기업 회원 / 통역사 회원 분리 가입
- 이메일 인증 후 JWT 발급

### 2. 통역사 자격 심사
- 이력서 파일 업로드 (PDF/이미지)
- 관리자 페이지에서 승인 / 반려 처리
- 승인 시 프로필 공개 전환

### 3. 검색 & 매칭
- 언어, 국가/도시, 전문 분야 필터
- trade show 날짜 기반 가용 여부 확인
- 기업 → 통역사 예약 요청 / 통역사 수락

### 4. 예약 & 결제
- 예약 확정 시 에스크로 방식 선결제
- 서비스 완료 후 통역사에게 정산 (수수료 차감)

### 5. 리뷰 / 평점
- 서비스 완료 후 양방향 평가
- 누적 평점 → 통역사 프로필 노출 순위 반영

## 개발 단계

### Phase 1 — MVP (1~2개월)
- 회원가입/로그인, 통역사 프로필 등록
- 관리자 자격 심사 기능
- 기본 검색 + 예약 요청

### Phase 2 — 결제 & 리뷰 (3개월)
- 결제 시스템 연동
- 리뷰/평점 기능
- 알림 (이메일/SMS)

### Phase 3 — 고도화 (4~6개월)
- 추천 알고리즘 (평점 + 전문 분야 매칭)
- 다국어 지원
- 모바일 최적화

## 보안 고려사항
- 개인정보(이력서, 연락처) 암호화 저장
- 결제 정보는 PG사 위임 (직접 저장 금지)
- 관리자 전용 페이지 접근 권한 분리
